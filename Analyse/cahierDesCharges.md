# Cahier des Charges
**Introduction :**

Ce document a pour objectif de définir et de clarifier les besoins clients afin de fournir un produit répondant aux mieux aux besoins clients.
Structure : Introduction, Enoncé, Pré-requis, Priorités.
Ce cahier des charges s'appuie sur les informations données par le document "sujet-de-la-sae3-2025.pdf".

**Enoncé :**

Ce projet à pour but de mettre en place une plateforme web permettant la gestion d’un parc informatique. Pour cela cette plateforme doit pouvoir proposer les services suivants :

**Objectif et portée :**

Quatre rôles sont à définir pour la gestion des droits de la plateforme pour permettre la gestion des utilisateurs.

**Les exigences fonctionnels à développer sont :**

* l'utilisateur peut consulter une partie de l'inventaire
* le technicien peut consulter le parc informatique
* le technicien peut modifier une information dans la liste du parc informatique/exporter une liste en format csv
* le technicien peut mettre une machine dans l’inventaire à partir d’un formulaire
* le technicien peut mettre une série de machines dans l’inventaire à partir d’un fichier de données
* le technicien peut supprimer une machine de l’inventaire pour la placer dans une liste dite du rebut/consulter liste rebut/changer statut matériel
* l'administrateur web peut créer un technicien dans la base
* l'administrateur web peut créer nom des systèmes d’exploitations/constructeur de la machine
* l'administrateur web peut supprimer un technicien
* l'administrateur web peut consulter la liste du rebut/bloquer la liste du rebut
* l'administrateur systeme peut consulter les différents journaux d’activités de la plateforme

**Pré-requis :** 

Connaissances requises : 

* Documentation du logiciel



Ressources matérielles et logicielles :
* raspBerry PI
* Serveur web
* Serveur SGBD
* Système d'exploitation

Compétences nécessaires :
* Aucune

**Priorités :**

La priorité est de produire un site web statique afin d'avoir un aperçu de l'interface de la platform.
